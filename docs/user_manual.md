# CheckMate ユーザーマニュアル

## 概要

CheckMate は、テストケースの作成、実行、履歴管理を効率的に行うためのデスクトップアプリケーションです。

**最新バージョン**: v1.1.2  
**対応プラットフォーム**: Windows, macOS, Linux

## 主な機能

1. テストスイート管理
2. テストケース作成・編集
3. テスト実行と結果記録
4. テスト履歴の閲覧
5. 前提条件の管理
6. テスト結果の PDF 書き出し
7. テスト結果の CSV エクスポート
8. 試験結果マージ機能

## ディレクトリ構成

アプリケーションは以下のディレクトリ構造を使用します：

```
CheckMate/
├── test_data/          # テストスイートのデータを保存
│   ├── [テストスイートID].json
│   └── image/          # テスト事前条件の画像を保存
│       └── [画像名].png/.jpeg
└── test_results/       # テスト実行結果を保存
    └── [テストスイート名]/
        └── [実行日]_[テスト名].json
```

### ディレクトリの説明

- test_data/: 各テストスイートの定義データが保存されます
  - テストケースの構成
  - 前提条件
  - テストステップ情報
- test_results/: テスト実行結果が保存されます
  - テストスイートごとにサブディレクトリを作成
  - 実行日とテスト名でファイルを管理
  - テスト結果（OK/NG/N/A）とコメントを保存

## 使い方

### 1. テストスイート管理

#### テストスイートの作成

1. メイン画面の「テストスイートを追加」ボタンをクリックします
2. テストスイート名を入力します
3. テスト管理番号を入力します（必須）
   - 既に存在するテスト管理番号は使用できません
   - 重複する管理番号を入力した場合、エラーメッセージが表示されます
4. 「追加」ボタンをクリックして保存します

#### テストスイートの編集・削除

- 名前変更：テストスイート一覧の「名前変更」ボタンをクリックします
- 削除：テストスイート一覧の「削除」ボタンをクリックします
  - 削除時は確認ダイアログが表示されます
  - 関連するテストケースとテスト結果も全て削除されます

#### テストスイートの検索

- 画面上部の検索ボックスにテストスイート名を入力すると、リアルタイムでフィルタリングされます

### 2. テストケース管理

テストスイート一覧からテストスイートをダブルクリックすると、テストケース管理画面に移動します。
画面上部のモードボタンで、以下の 3 つのモードを切り替えることができます：

#### ナビゲーション機能

- **スティッキーヘッダー**：画面をスクロールしても、ヘッダー部分（タイトル、モード切り替えボタンなど）が常に上部に固定表示されます
- **一番上に戻るボタン**：タイトル横の「↑」ボタンをクリックすると、画面の最上部にスムーズにスクロールします
- 長いテストケースリストでも、常にナビゲーションにアクセスできるため効率的な操作が可能です

#### 前提条件の管理

テストケース画面上部に前提条件を表示・編集できます。前提条件は以下の4つの項目で構造化して管理されます：

##### 前提条件の表示
- **項目別表示**：目的・環境・使用ツール・事前準備の4項目を個別表示
- **統合表示**：従来のMarkdown形式で統合して表示
- 右上の切り替えボタンで表示モードを変更可能
- 折りたたみ機能で表示領域を調整可能

##### 前提条件の編集
- 「編集」ボタンをクリックして編集画面を開く
- **アコーディオン形式**の編集UI：
  - **目的**：テストの目的や実現したい目標
  - **環境**：テスト実行に必要な環境や条件
  - **使用ツール**：テストで使用するツールやソフトウェア
  - **事前準備**：テスト実行前に行う準備や設定（縦幅が大きく、リサイズ可能）
- 各項目は独立してMarkdown記法で記述可能
- リアルタイムプレビュー機能付き
- 統合プレビューで全体の表示を確認可能

##### レガシーデータの対応
- 既存のMarkdown形式の前提条件は自動で4項目に分離
- セクションヘッダー（「## 目的」「## 環境」など）を検出して自動変換
- 分離できない内容は「事前準備」項目に配置
- 下位互換性を完全保持

##### 画像表示機能
- 前提条件では画像も表示可能
- 画像は test_data/image ディレクトリを作成し、image ディレクトリに入れてください
- markdown で exe のディレクトリからの相対パスを入力すると表示されます

#### 編集モード

- 新規テストケースの追加
  - 「テストケースを追加」ボタンをクリック
  - デフォルトで「新しいテストケース」という名前で作成
  - 手順と期待値の入力欄が自動的に追加
- テストケースの編集
  - テストケース名の変更
  - テストステップの追加・編集・削除
    - テストケース行で右クリック：テストケースの挿入/削除
    - テストステップ行で右クリック：テストステップの挿入/削除、テンプレートの挿入
  - 期待結果の設定
  - 各ステップに対して手順と期待値を詳細に記述可能
  - テンプレートの挿入
    - テストステップを右クリックして「テンプレートを挿入」を選択
    - 選択したテンプレートのステップが、選択位置の後に挿入されます
- テストケースの複製
  - テストケース行で右クリック：「テストケースを複製」を選択
  - 複製したテストケースの挿入位置を選択
    - 先頭に挿入
    - 既存のテストケースの後に挿入（複製元のテストケース以外）
  - 選択したテストケースと同じ内容の新しいテストケースが、指定した位置に作成されます
- テストケースの折りたたみ
  - 各テストケースの左側の「+/-」ボタンで個別に折りたたみ可能
  - 「すべて折りたたむ/展開」ボタンで一括操作可能
  - テストケースごとに背景色が異なり、視認性が向上

#### テストモード

- テストケースの実行
- テスト結果の記録（OK,NG,N/A）
- テスト実行時のメモ追加
- 前提条件を参照しながらテスト実行が可能
- テスト結果の自動保存（10 秒間隔）
- テスト実施状況の集計表示
  - テストケース単位での結果を集計（OK/NG/N/A/未実施の数）
  - テストケースの判定基準：
    - 未実施：1つでも未実施のステップがある場合
    - NG：1つでもNGのステップがある場合
    - OK：すべてのステップがOKまたはN/Aで、少なくとも1つのOKがある場合
    - N/A：すべてのステップがN/Aの場合

#### 閲覧モード

- 過去のテスト実行履歴の確認
- テスト結果の詳細表示
  - テストケース単位での結果集計（OK/NG/N/A/未実施の数）
  - 各テストケースの詳細なステップ結果
  - テストケースごとの判定結果（テストモードと同じ判定基準を使用）
- テストケース単位の集計テーブル
  - 折りたたみ可能なテーブルで表示
  - テストケースごとに以下の情報を表示：
    - テストケース番号（No.）
    - テストケース名
    - OK数（緑色で表示）
    - NG数（赤色で表示）
    - N/A数（黄色で表示）
  - テーブルは▼/▶ボタンで開閉可能
  - デフォルトで開いた状態で表示
- **履歴表示機能の強化**
  - テストケースごとに異なる背景色で表示（視認性向上）
  - テストケース名は各テストケースの最初のステップにのみ表示
  - ステップ番号は「テストケース番号-ステップ番号」形式で表示
  - 実行結果（OK/NG/N/A）とコメントを明確に表示
- 日付別のテスト結果確認
- PDF出力機能（下記「### 5. PDF 書き出し機能」参照）
- CSVエクスポート機能（下記「### 6. CSV エクスポート機能」参照）

### 3. 自動保存機能

#### 自動保存の設定

画面上部に自動保存のON/OFFを切り替えるトグルスイッチがあります：

- デフォルトでは自動保存が有効（ON）になっています
- トグルスイッチをクリックすることで、自動保存を無効（OFF）にできます
- この設定は編集モード・テストモード両方に適用されます

#### 自動保存のタイミング

自動保存が有効な場合、以下のタイミングで保存が実行されます：

- 編集開始から 10 秒経過時
- テスト結果の記録時
- 前提条件の変更時

保存が完了すると、デスクトップ通知で通知されます。

### 4. ユーザー設定

#### ユーザー名の設定

1. メイン画面右上の「変更」ボタンをクリックします
2. 新しいユーザー名を入力します
3. 「変更」ボタンをクリックして保存します

## ショートカットキー

- テストスイート一覧に戻る：戻るボタンをクリック
- テストケース画面を開く：テストスイートをダブルクリック
- テストケース画面のショートカット
  - Ctrl+S : 保存
  - Ctrl+Shift+E: 編集モードに切り替え
  - Ctrl+Shift+T: テストモードに切り替え
  - Ctrl+Shift+W: 閲覧モードに切り替え
  - Ctrl+Tab: モードを循環的に切り替え（edit → test → history → edit）
- ナビゲーション機能
  - 一番上に戻るボタン（↑）：ページトップへスムーズにスクロール

## 注意事項

1. テストスイートの削除は取り消しできません
2. テスト結果は日付単位で保存されます
3. 長時間の編集作業を行う場合は、定期的に手動保存することを推奨します
4. テスト管理番号は必須項目となっています
5. 前提条件の変更も自動保存の対象となります
6. test_data ディレクトリに余分なファイルがある場合や、手動で編集して誤った json 形式のファイルが保存されると、起動時の読み込みに失敗します
7. markdown のテーブル編集機能はありません

### 5. PDF 書き出し機能

#### テスト結果の PDF 書き出し

1. テストケース画面で閲覧モードに切り替えます
2. 画面右上の「PDF 出力」ボタンをクリックします
3. 保存先を選択して PDF ファイルを保存します

PDF には以下のページが含まれます：

1. **表紙ページ**：
   - テストスイート名（大きめのフォントで表示）

2. **集計ページ**：
   - 全体の集計（OK/NG/N/A/未実施の数）
   - テストケース単位の集計テーブル
     - テストケース番号（No.）：テストケースの状態に応じて背景色が変化
       * OK（緑色）：OKのステップが1つ以上ある
       * NG（赤色）：NGのステップが1つでもある
       * N/A（オレンジ色）：すべてのステップがN/Aの場合
     - テストケース名
     - 各テストケースのOK数
     - 各テストケースのNG数
     - 各テストケースのN/A数

3. **前提条件ページ**（設定されている場合のみ）：
   - テストスイートで設定された前提条件を表示
   - Markdown記法（太字、箇条書きなど）も適切に表示される
   - 画像参照も含めて表示される

4. **テスト結果詳細ページ**：
   - テストケース一覧
   - 各テストケースの詳細なステップ結果
     - テストケース名
     - No（テストケース番号-ステップ番号の形式。例：1-1, 1-2, 2-1など）
     - 手順
     - 期待値
     - 結果（OK/NG/N/A）
     - コメント

#### 注意事項

- 日本語フォントが使用されているため、日本語テキストも適切に表示されます
- Markdown記法（**太字**、*斜体*、箇条書きなど）はPDFでも適切にフォーマットされます
- テスト結果の前提条件は集計ページと詳細ページの間に表示されます

## トラブルシューティング

### PDF 出力ができない場合

1. 閲覧モードになっていることを確認してください
2. テスト結果が正しく保存されているか確認してください
3. 保存先のディレクトリに書き込み権限があることを確認してください

### CSV エクスポートができない場合

1. 閲覧モードになっていることを確認してください
2. テスト結果が正しく選択されているか確認してください
3. 保存先のディレクトリに書き込み権限があることを確認してください
4. ファイル名に使用できない文字が含まれていないか確認してください

### 保存できない場合

1. アプリケーションを再起動してください
2. 保存先のディレクトリに書き込み権限があることを確認してください
3. test_data ディレクトリと test_results ディレクトリが存在することを確認してください

### テスト結果が表示されない場合

1. テストモードに切り替えてください
2. テスト結果ファイルが正しく保存されているか確認してください
3. test_results/[テストスイート名]/ ディレクトリ内にファイルが存在するか確認してください

### 自動保存が機能しない場合

1. デスクトップ通知の権限が許可されているか確認してください
2. 編集内容に変更があるか確認してください
3. test_data ディレクトリへの書き込み権限があるか確認してください

### 6. CSV エクスポート機能

#### CSV エクスポートの概要

閲覧モードでテスト結果をCSV形式で出力することができます。
Excel等の表計算ソフトで開いて分析や加工を行うことができます。

#### CSV エクスポートの手順

1. テストケース画面で閲覧モードに切り替えます
2. 画面右上の「CSVエクスポート」ボタンをクリックします
3. 保存先を選択してCSVファイルを保存します

#### CSV の出力内容

CSVファイルには以下の情報が論理的な順序で含まれます：

1. **試験名称**：テストスイート名
2. **前提条件**：設定されている場合のみ表示（試験名称の直後に配置）
3. **テスト結果集計**：実行済みテストがある場合のみ表示
   - OK数
   - NG数  
   - N/A数
   - 未実施数
4. **テスト詳細**：以下の列で構成されるテーブル
   - **テストケース**：テストケース名（各テストケースの最初の行のみ表示、Excel互換性向上のためシングルクォート付き）
   - **No**：テストケース番号-ステップ番号（例：1-1, 1-2, 2-1など）
   - **手順**：テストステップの内容
   - **期待値**：期待される結果
   - **結果**：実行結果（OK/NG/N/A）
   - **コメント**：テスト実行時のコメント

#### 注意事項

- 文字化けを防ぐためBOM付きUTF-8で出力されます
- Markdownの改行は適切に処理されてCSV内で保持されます
- ダブルクォートは自動的にエスケープされます
- 閲覧モードでのみ利用可能です
- 未実行のテストスイートでは集計セクションは表示されません
- Excelでテストケース名が数値として誤解釈されることを防ぐため、シングルクォートが自動追加されます

### 7. 試験結果マージ機能

#### マージ機能の概要

同一テストスイート内の2つの試験結果を1つのファイルにマージすることができます。
結果が重複している場合は、どちらの結果を採用するか選択することができます。

#### マージの手順

1. テストケース画面で「結果マージ」ボタンをクリックします
2. マージしたい2つの結果ファイルをドロップダウンから選択します
3. 結果が重複している場合：
   - 重複している項目が表示されます
   - 各項目について、どちらの結果を採用するか選択します
   - すべての重複が解消されるまでマージは実行できません
4. 出力ファイル名を入力します
5. 「マージ実行」ボタンをクリックして保存します

#### 注意事項

- 同一テストスイート内の結果のみマージ可能です
- 重複している項目は以下の要素をチェックします：
  - 試験結果（OK/NG/N/A）
  - コメント
- すべての重複項目の採用結果を選択するまでマージできません
- マージ結果は新しいファイルとして保存されます

### 8. テンプレート機能

#### テンプレート一覧画面

1. メイン画面の「テンプレート一覧」をクリックします
2. 作成済みのテンプレート一覧が表示されます
3. 各テンプレートには以下の情報が表示されます：
   - テンプレート名
   - 説明
   - ステップ数
4. テンプレートの編集・削除が可能です

#### テンプレートの作成

1. テンプレート一覧画面で「新規作成」ボタンをクリックします
2. テンプレート名を入力します
3. 説明を入力します（任意）
4. テストステップを追加します
   - 「手順を追加」ボタンで新しいステップを追加
   - 各ステップに手順と期待値を入力
   - ステップの挿入・削除も可能
5. 「保存」ボタンをクリックして保存します

#### テンプレートの挿入

1. テストケース編集画面でテストステップを右クリックします
2. コンテキストメニューから「テンプレートを挿入」を選択します
3. テンプレート選択モーダルが表示されます
4. 挿入したいテンプレートをクリックします
5. 選択したテンプレートのステップが、選択位置の後に挿入されます

### v1.1.2 の新機能・改善点

#### CSVエクスポート機能の強化
- 出力形式の改善：前提条件を試験名称の直後に配置
- テスト結果集計の表示条件改善：実行済みテストがある場合のみ表示
- Excel互換性向上：テストケース名にシングルクォート追加で数値誤認識を防止

#### PDF出力機能の強化
- PDFエクスポート機能を独立したコンポーネントに分離
- 前提条件ページの追加（設定されている場合）
- 日本語フォント対応とMarkdown記法の適切な変換
- 保守性とコードの再利用性を大幅に向上

#### UI・表示機能の改善
- 履歴モードでのステップ表示機能強化
- テストケースごとの背景色による視認性向上
- ステップ表示の統一化と共通化

#### 技術基盤の更新
- React 19への移行
- 依存関係の大幅なアップデート
- Tauriプラグインの最新化

### v1.1.1 からの主な変更点
- CSVエクスポート機能の追加
- PDF出力時の前提条件表示対応
- コンポーネントの設計改善とリファクタリング
