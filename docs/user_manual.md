# CheckMate ユーザーマニュアル

## 概要

CheckMateは、テストケースの作成、実行、履歴管理を効率的に行うためのデスクトップアプリケーションです。

## 主な機能

1. テストスイート管理
2. テストケース作成・編集
3. テスト実行と結果記録
4. テスト履歴の閲覧
5. 前提条件の管理

## ディレクトリ構成

アプリケーションは以下のディレクトリ構造を使用します：

```
CheckMate/
├── test_data/          # テストスイートのデータを保存
│   ├── [テストスイートID].json
│   └── image/          # テスト事前条件の画像を保存
│       └── [画像名].png/.jpeg
└── test_results/       # テスト実行結果を保存
    └── [テストスイート名]/
        └── [実行日]_[テスト名].json
```

### ディレクトリの説明
- test_data/: 各テストスイートの定義データが保存されます
  - テストケースの構成
  - 前提条件
  - テストステップ情報
- test_results/: テスト実行結果が保存されます
  - テストスイートごとにサブディレクトリを作成
  - 実行日とテスト名でファイルを管理
  - テスト結果（OK/NG/N/A）とコメントを保存

## 使い方

### 1. テストスイート管理

#### テストスイートの作成
1. メイン画面の「テストスイートを追加」ボタンをクリックします
2. テストスイート名を入力します
3. テスト管理番号を入力します（必須）
4. 「追加」ボタンをクリックして保存します

#### テストスイートの編集・削除
- 名前変更：テストスイート一覧の「名前変更」ボタンをクリックします
- 削除：テストスイート一覧の「削除」ボタンをクリックします
  - 削除時は確認ダイアログが表示されます
  - 関連するテストケースとテスト結果も全て削除されます

#### テストスイートの検索
- 画面上部の検索ボックスにテストスイート名を入力すると、リアルタイムでフィルタリングされます

### 2. テストケース管理

テストスイート一覧からテストスイートをダブルクリックすると、テストケース管理画面に移動します。
画面上部のモードボタンで、以下の3つのモードを切り替えることができます：

#### 前提条件の管理
- テストケース画面上部に前提条件を表示
- 「編集」ボタンをクリックして前提条件を編集
- テストスイート全体に適用される共通の前提条件を設定可能
- 前提条件では画像も表示可能
  - 画像はtest_data/imageディレクトリを作成し、imageディレクトリに入れてください
  - markdownでexeのディレクトリからの相対パスを入力すると表示されます

#### 編集モード
- 新規テストケースの追加
  - 「テストケースを追加」ボタンをクリック
  - デフォルトで「新しいテストケース」という名前で作成
  - 手順と期待値の入力欄が自動的に追加
- テストケースの編集
  - テストケース名の変更
  - テストステップの追加・編集・削除
  - 期待結果の設定
  - 各ステップに対して手順と期待値を詳細に記述可能

#### テストモード
- テストケースの実行
- テスト結果の記録（OK/NG/N/A）
- テスト実行時のメモ追加
- 前提条件を参照しながらテスト実行が可能

#### 閲覧モード
- 過去のテスト実行履歴の確認
- テスト結果の詳細表示
- 日付別のテスト結果確認

### 3. 自動保存機能

テストケースの編集内容は以下のタイミングで自動保存されます：
- 編集開始から10秒経過時
- テスト結果の記録時
- 前提条件の変更時

保存が完了すると、デスクトップ通知で通知されます。

### 4. ユーザー設定

#### ユーザー名の設定
1. メイン画面右上の「変更」ボタンをクリックします
2. 新しいユーザー名を入力します
3. 「変更」ボタンをクリックして保存します

## ショートカットキー

- テストスイート一覧に戻る：戻るボタンをクリック
- テストケース画面を開く：テストスイートをダブルクリック
- テストケース画面のショートカット
  - Ctrl+S      : 保存
  - Ctrl+Shift+E: 編集モードに切り替え
  - Ctrl+Shift+T: テストモードに切り替え
  - Ctrl+Shift+W: 閲覧モードに切り替え
  - Ctrl+Tab: モードを循環的に切り替え（edit → test → history → edit）

## 注意事項

1. テストスイートの削除は取り消しできません
2. テスト結果は日付単位で保存されます
3. 長時間の編集作業を行う場合は、定期的に手動保存することを推奨します
4. テスト管理番号は必須項目となっています
5. 前提条件の変更も自動保存の対象となります
6. test_dataディレクトリに余分なファイルがある場合や、手動で編集して誤ったjson形式のファイルが保存されると、起動時の読み込みに失敗します
7. markdownのテーブル編集機能はありません

## トラブルシューティング

### 保存できない場合
1. アプリケーションを再起動してください
2. 保存先のディレクトリに書き込み権限があることを確認してください
3. test_dataディレクトリとtest_resultsディレクトリが存在することを確認してください

### テスト結果が表示されない場合
1. テストモードに切り替えてください
2. テスト結果ファイルが正しく保存されているか確認してください
3. test_results/[テストスイート名]/ ディレクトリ内にファイルが存在するか確認してください

### 自動保存が機能しない場合
1. デスクトップ通知の権限が許可されているか確認してください
2. 編集内容に変更があるか確認してください
3. test_dataディレクトリへの書き込み権限があるか確認してください
