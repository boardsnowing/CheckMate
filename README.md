# CheckMate
## 概要

CheckMateはテストケースの作成・実行・管理を効率的に行うデスクトップアプリケーションです。  
これまでexcelで作成されていた検査要領書をアプリケーションで管理するためのツールとなります。

Tauri+Reactで構築されたデスクトップ向けテスト管理ツールです。  
テストスイートの管理からテスト実行・結果記録・PDF/CSV出力まで、テスト業務を一元管理します。

作成したテストファイルおよびテスト実行結果はjson形式で保存され、gitフレンドリーです。

## 主な機能

- **テストスイート管理** — 複数のテストスイートを作成・編集・削除
- **テストケース管理** — テストステップの作成・編集・複製・折りたたみ表示
- **前提条件の構造化管理** — 目的・環境・使用ツール・事前準備の4項目で整理
- **テスト実行・結果記録** — OK/NG/N/A の記録とコメント入力、自動保存（10秒間隔）
- **テンプレート機能** — 再利用可能なテストステップのテンプレート管理
- **共通手順機能** — テストケース間で共有される手順をグループとして挿入・管理
- **PDF書き出し** — 表紙・集計・前提条件・詳細の4部構成でPDF出力
- **CSVエクスポート** — BOM付きUTF-8でExcel対応のCSV出力
- **閲覧モード** — 過去のテスト実行履歴の確認と集計表示

## セットアップ

### 前提条件

- Node.js 18以上
- Rust（最新安定版）
- Tauri CLI v2

### インストール

```bash
npm install
```

### 開発サーバー起動

```bash
npm run tauri dev
```

### プロダクションビルド

```bash
npm run tauri build
```

## データ構造

CheckMate のデータはテストスイート→前提条件+テストケース→テストステップの3階層で管理されます。

```
テストスイート（検査要領書 1冊に相当）
├── 前提条件（目的 / 環境 / 使用ツール / 事前準備）
├── テストケース 1（検査項目 1つに相当）
│   ├── テストステップ 1-1
│   ├── テストステップ 1-2
│   └── ...
├── テストケース 2
│   ├── テストステップ 2-1
│   └── ...
└── ...
```

### テストスイート

テストの集合体です。Excelの1ファイル（検査要領書）に相当します。  
テスト管理番号（重複不可）とスイート名で識別します。  
テストスイートごとに**前提条件**（目的・環境・使用ツール・事前準備）を設定できます。

### テストケース

テストスイート内の個別の検査項目です。Excelの1シート（または1シナリオ）に相当します。  
複数のテストステップで構成され、テンプレートや**共通手順**を使って効率的に作成できます。  
テスト実行後は、含まれる全ステップの結果からケース単位の判定（OK/NG/N/A/未実施）が自動算出されます。

### テストステップ

テストケース内の1操作です。「手順」と「期待値」の組み合わせで定義します。  
実行時に **OK / NG / N/A** を記録し、任意でコメントを残せます。  
複数のテストケースで共通して使う操作は**共通手順**としてまとめて管理でき、変更が全参照先に自動反映されます。
